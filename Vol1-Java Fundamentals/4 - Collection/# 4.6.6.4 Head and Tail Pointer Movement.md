# 4.6.6.4 Head and Tail Pointer Movement

The performance of `ArrayDeque` comes from a simple but powerful idea:

> **Most operations only move the `head` or `tail` pointer.**

Unlike arrays, no element shifting is required.

Unlike linked lists, no node allocation or pointer manipulation is required.

The implementation simply updates one or two integer indices.

Understanding how these indices move is the key to understanding the entire data structure.

---

# Initial State

Consider an empty deque with a capacity of 8.

```text
Index

 0    1    2    3    4    5    6    7

+----+----+----+----+----+----+----+----+

|    |    |    |    |    |    |    |    |

+----+----+----+----+----+----+----+----+

head = 0

tail = 0
```

The deque is empty because:

```text
head == tail
```

---

# addLast()

Suppose we execute:

```java
deque.addLast("A");
```

The element is written at the current `tail` position.

```text
Before

head = 0

tail = 0
```

```text
Index

0 1 2 3 4 5 6 7

+---+---+---+---+---+---+---+---+

| A |   |   |   |   |   |   |   |

+---+---+---+---+---+---+---+---+

head = 0

tail = 1
```

Notice the sequence:

1. Store the element at `tail`.
2. Advance `tail`.

The newly inserted element becomes the last logical element.

The `tail` always points to the **next available insertion position**.

---

# Multiple addLast() Operations

Execute:

```java
deque.addLast("B");

deque.addLast("C");

deque.addLast("D");
```

Result:

```text
Index

0 1 2 3 4 5 6 7

+---+---+---+---+---+---+---+---+

| A | B | C | D |   |   |   |   |

+---+---+---+---+---+---+---+---+

head = 0

tail = 4
```

The deque grows toward the right.

---

# removeFirst()

Now execute:

```java
deque.removeFirst();
```

Before:

```text
A B C D
```

After:

```text
B C D
```

Internally:

```text
Index

0 1 2 3 4 5 6 7

+---+---+---+---+---+---+---+---+

|   | B | C | D |   |   |   |   |

+---+---+---+---+---+---+---+---+

head = 1

tail = 4
```

Important observation:

The elements:

```text
B C D
```

remain exactly where they were.

Only `head` changes.

---

# addFirst()

Suppose we now execute:

```java
deque.addFirst("X");
```

Unlike `addLast()`, insertion occurs before the current `head`.

Before insertion:

```text
head = 1
```

The implementation first moves `head` backward.

Then it stores the element.

Result:

```text
Index

0 1 2 3 4 5 6 7

+---+---+---+---+---+---+---+---+

| X | B | C | D |   |   |   |   |

+---+---+---+---+---+---+---+---+

head = 0

tail = 4
```

The logical order becomes:

```text
X

↓

B

↓

C

↓

D
```

---

# removeLast()

Suppose we remove the rear element.

```java
deque.removeLast();
```

Current state:

```text
X B C D
```

The implementation performs:

1. Move `tail` backward.
2. Read the element.
3. Clear the array slot.

Result:

```text
Index

0 1 2 3 4 5 6 7

+---+---+---+---+---+---+---+---+

| X | B | C |   |   |   |   |   |

+---+---+---+---+---+---+---+---+

head = 0

tail = 3
```

Again:

No shifting.

Only pointer movement.

---

# Pointer Movement Summary

| Operation | Head | Tail |
|-----------|------|------|
| addFirst() | Moves backward | Unchanged |
| addLast() | Unchanged | Moves forward |
| removeFirst() | Moves forward | Unchanged |
| removeLast() | Unchanged | Moves backward |

This symmetry makes the implementation simple and efficient.

# Wrap-Around Example

Suppose:

```text
head = 6

tail = 2
```

Physical array:

```text
Index

0 1 2 3 4 5 6 7

F G . . . . A B
```

Logical order:

```text
A

↓

B

↓

F

↓

G
```

Now execute:

```java
addLast("H");
```

`tail` is currently:

```text
2
```

The new element is stored at index 2.

```text
Index

0 1 2 3 4 5 6 7

F G H . . . A B
```

The logical order becomes:

```text
A

↓

B

↓

F

↓

G

↓

H
```

Although the physical storage appears unusual, the logical sequence remains correct.

---

# Why Pointer Movement Is Fast

Updating an integer index is one of the cheapest operations a CPU can perform.

Example:

```java
head++;

tail--;
```

Compared with:

```text
Copy

999,999 Elements
```

the cost is negligible.

This is why `ArrayDeque` scales efficiently even under heavy workloads.

---

# Internal State After Each Operation

| Operation | Head | Tail | Logical Contents |
|-----------|------|------|------------------|
| Empty | 0 | 0 | — |
| addLast(A) | 0 | 1 | A |
| addLast(B) | 0 | 2 | A B |
| addLast(C) | 0 | 3 | A B C |
| removeFirst() | 1 | 3 | B C |
| addFirst(X) | 0 | 3 | X B C |
| removeLast() | 0 | 2 | X B |

Notice that only the indices change.

The majority of elements never move.

---

# Common Misconceptions

## "`head` always stays at index 0."

Incorrect.

`head` continuously moves as elements are inserted and removed.

---

## "`tail` always points to the last element."

No.

It points to the next insertion position.

This simplifies insertion and wrap-around logic.

---

## "`removeLast()` deletes the last array element."

Not necessarily.

It removes the last **logical** element, which may be anywhere in the circular array depending on the current positions of `head` and `tail`.

---

# Interview Questions

### Q1. Which pointer changes during `addFirst()`?

`head` moves backward, then the new element is inserted at the updated position.

---

### Q2. Which pointer changes during `removeFirst()`?

Only `head` advances to the next logical element.

---

### Q3. Which pointer changes during `addLast()`?

Only `tail` advances after storing the new element.

---

### Q4. Why does `tail` represent the next insertion position instead of the last element?

This design simplifies insertion operations and avoids additional calculations when adding elements to the rear.

---

# Senior Interview Perspective

A common interview discussion is:

> **Interviewer:** "Walk me through the internal state of an `ArrayDeque` after performing `addLast(A)`, `addLast(B)`, `removeFirst()`, and `addFirst(C)`."

A strong answer is:

> "Each operation updates either the `head` or `tail` index without shifting existing elements. `addLast()` writes at the current `tail` and advances it, `removeFirst()` advances the `head`, and `addFirst()` moves the `head` backward before inserting the new element. The logical order changes, but the underlying array contents remain largely untouched."

This demonstrates a solid understanding of the implementation rather than just the API.

---

# Key Takeaways

- `ArrayDeque` operations are implemented primarily through pointer movement.
- `head` identifies the first logical element, while `tail` identifies the next insertion position at the rear.
- Insertions and removals generally update only one pointer, avoiding element shifting.
- The symmetry of pointer movement simplifies support for both queue and stack operations.
- Understanding `head` and `tail` movement is essential before exploring wrap-around logic, resizing, and the underlying OpenJDK implementation.
