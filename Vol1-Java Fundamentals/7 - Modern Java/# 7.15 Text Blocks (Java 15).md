# 7.15 Text Blocks (Java 15)

**Text Blocks**, introduced as a standard feature in **Java 15**, provide a clean and readable way to write **multi-line string literals**.

Before Text Blocks, developers had to concatenate multiple strings, manually insert newline characters (`\n`), and escape quotation marks.

Text Blocks eliminate this boilerplate, making code easier to write and maintain.

They are particularly useful for:

- SQL queries
- JSON
- XML
- HTML
- GraphQL
- YAML
- Email templates
- Multi-line log messages

---

# Why Were Text Blocks Introduced?

Consider a SQL query before Java 15.

```java
String sql =
    "SELECT id, name " +
    "FROM employee " +
    "WHERE salary > 50000 " +
    "ORDER BY name";
```

Or, if formatting is desired:

```java
String sql =
    "SELECT id, name\n" +
    "FROM employee\n" +
    "WHERE salary > 50000\n" +
    "ORDER BY name";
```

Problems:

- Hard to read
- Lots of concatenation
- Manual `\n`
- Difficult to maintain

Text Blocks solve these issues.

---

# Basic Syntax

A Text Block starts and ends with three double quotes (`"""`).

Example:

```java
String message = """
Hello Java
Welcome to Spring Boot
""";
```

Output:

```text
Hello Java
Welcome to Spring Boot
```

No concatenation.

No newline characters.

---

# SQL Example

Without Text Blocks:

```java
String sql =
    "SELECT * FROM employee " +
    "WHERE department = ?";
```

With Text Blocks:

```java
String sql = """
        SELECT *
        FROM employee
        WHERE department = ?
        """;
```

Much easier to read.

---

# JSON Example

Before Java 15:

```java
String json =
"{\"id\":1,\"name\":\"Alice\"}";
```

With Text Blocks:

```java
String json = """
{
    "id": 1,
    "name": "Alice"
}
""";
```

No escaped quotes.

# HTML Example

```java
String html = """
```

Perfect for HTML templates.


Compared to string concatenation, this is significantly more maintainable.

# Indentation

The compiler automatically removes common indentation.

Example:

```java
String text = """
        Java
        Spring
        Kafka
        """;
```

Output:

```text
Java
Spring
Kafka
```

The source code remains properly indented without affecting the resulting string.

---

# Trailing Spaces

Trailing spaces are ignored by default.

Example:

```java
"""
Java
"""
```

Produces:

```text
Java
```

without preserving unnecessary whitespace.

---

# Escaping Characters

Most escaping is unnecessary.

Example:

```java
String text = """
He said "Hello"
""";
```

Output:

```text
He said "Hello"
```

Quotation marks do not need escaping.

---

# Escape Sequences Still Work

Example:

```java
String text = """
Hello\tJava
""";
```

Output:

```text
Hello    Java
```

Standard escape sequences such as:

- `\n`
- `\t`
- `\\`

continue to work.

---

# Including Triple Quotes

Occasionally triple quotes must appear inside a text block.

Example:

```java
String text = """
He wrote \"""
Java
\"""
""";
```

Escaping is still available when required.

---

# String Methods

A Text Block is still a `String`.

Example:

```java
String text = """
Java
Spring
""";
```

You can call:

```java
text.length();

text.contains("Java");

text.toUpperCase();

text.replace(...);
```

Exactly like any other string.

---

# Text Blocks vs String Concatenation

Traditional:

```java
String sql =
    "SELECT * " +
    "FROM employee";
```

Text Block:

```java
String sql = """
SELECT *
FROM employee
""";
```

Advantages:

- Better readability
- Less boilerplate
- Easier maintenance

---

# Text Blocks vs StringBuilder

`StringBuilder` is used when strings are constructed dynamically.

Example:

```java
StringBuilder builder =
        new StringBuilder();

builder.append("Hello");

builder.append(name);
```

Text Blocks are intended for **static multi-line literals**, not dynamic string construction.

# Performance

Text Blocks are **compile-time string literals**.

The generated bytecode is effectively equivalent to using ordinary string literals.

There is **no runtime performance penalty** for using Text Blocks.

Their primary benefit is improved readability and maintainability.

---

# Best Practices

- Use Text Blocks for static multi-line content.
- Prefer Text Blocks for SQL, JSON, XML, and HTML.
- Keep indentation consistent for readability.
- Avoid unnecessary escape sequences.
- Use `StringBuilder` when constructing strings dynamically.

---

# Common Mistakes

## Expecting Automatic Variable Interpolation

Java Text Blocks do **not** support variable interpolation.

Incorrect:

```java
String text = """
Hello ${name}
""";
```

Output:

```text
Hello ${name}
```

The placeholder is treated as ordinary text.

Use `String.format()` or the `formatted()` method instead.

Example:

```java
String text = """
Hello %s
""".formatted(name);
```

---

## Using Text Blocks for Single-Line Strings

Avoid:

```java
String text = """
Java
""";
```

when a normal string literal is sufficient.

Prefer:

```java
String text = "Java";
```

for simple single-line values.

---

## Using Text Blocks for Dynamic Content

Avoid concatenating dynamic values inside a Text Block.

Instead:

```java
String sql = """
SELECT *
FROM employee
WHERE id = %d
""".formatted(id);
```

or use prepared statements for SQL.

# Interview Questions

### Q1. Why were Text Blocks introduced?

To simplify writing multi-line string literals by eliminating manual concatenation, newline characters, and unnecessary escaping.

---

### Q2. Are Text Blocks mutable?

No.

A Text Block is still a `String`, and `String` remains immutable.

---

### Q3. Do quotation marks need to be escaped inside a Text Block?

No.

Ordinary double quotes can be written directly without escaping.

---

### Q4. Are Text Blocks faster than ordinary strings?

No.

They are compile-time string literals and produce essentially the same bytecode as traditional string literals.

Their benefit is improved readability rather than performance.

---

### Q5. What are common use cases for Text Blocks?

- SQL queries
- JSON payloads
- XML documents
- HTML templates
- GraphQL queries
- Email templates
- Configuration snippets

# Key Takeaways

- Text Blocks provide a clean syntax for multi-line string literals.
- They eliminate manual concatenation and reduce the need for escape sequences.
- Text Blocks are ideal for SQL, JSON, XML, HTML, GraphQL, and other structured text.
- They remain ordinary immutable `String` objects with no runtime performance overhead.
- Use Text Blocks for readability, but continue using `StringBuilder` for dynamic string construction.
- Combined with other modern Java features, Text Blocks contribute to cleaner and more maintainable enterprise code.
