# Chapter 7 – Modern Java (Java 8–21)

## 7.1 Evolution of Java

Java has evolved significantly since Java 8, introducing language features, APIs, and JVM improvements that make applications more expressive, performant, and maintainable.

For a senior backend engineer, understanding **why a feature was introduced**, **when to use it**, and **its performance implications** is far more important than simply memorizing syntax.

This chapter covers the most impactful features from Java 8 through Java 21, focusing on real-world backend development and interview expectations.

## Timeline of Major Java Releases

| Java Version | Major Features |
|--------------|----------------|
| Java 8 | Lambda Expressions, Stream API, Optional, Date & Time API |
| Java 9 | Module System (JPMS), JShell, Collection Factory Methods |
| Java 10 | Local Variable Type Inference (`var`) |
| Java 11 | Standard HTTP Client, String API enhancements, LTS Release |
| Java 12–14 | Switch Expressions, Helpful NullPointerExceptions (preview/final) |
| Java 15 | Text Blocks |
| Java 16 | Records |
| Java 17 | Sealed Classes, Pattern Matching improvements, LTS Release |
| Java 19–20 | Virtual Threads (Preview), Structured Concurrency (Preview) |
| Java 21 | Virtual Threads (Final), Record Patterns, Pattern Matching for `switch`, Sequenced Collections, LTS Release |

## Why Modern Java Matters

Older Java code often looks like this:

```java
List<String> names = new ArrayList<String>();

for (String name : names) {
    System.out.println(name);
}
```

Modern Java allows writing the same intent more clearly:

```java
List<String> names = List.of("Alice", "Bob", "Charlie");

names.forEach(System.out::println);
```

Modern language features improve:

- Readability
- Developer productivity
- Safety
- Performance (in many scenarios)
- Maintainability
