# 7.2 Lambda Expressions

One of the biggest changes introduced in **Java 8** was the addition of **Lambda Expressions**.

Before Java 8, Java was primarily an object-oriented language where behavior was represented using classes and objects.

With Java 8, Java also embraced **functional programming concepts**, allowing behavior to be treated as data.

Lambda expressions make code:

- More concise
- More readable
- Easier to maintain
- Better suited for parallel processing

Today, they are used extensively throughout the Java ecosystem, especially in:

- Stream API
- Collections Framework
- CompletableFuture
- Spring Framework
- Event-driven programming

---

# Why Were Lambda Expressions Introduced?

Before Java 8, passing behavior required creating anonymous classes.

Example:

```java
List<String> names = Arrays.asList("Alice", "Bob", "Charlie");

Collections.sort(names, new Comparator<String>() {
    @Override
    public int compare(String a, String b) {
        return a.compareTo(b);
    }
});
```

Although simple, this code contains a lot of boilerplate.

The actual business logic is only:

```java
a.compareTo(b)
```

Everything else exists only because Java required an anonymous class.

Lambda expressions eliminate this unnecessary code.

---

# The Same Code Using Lambda

```java
List<String> names = Arrays.asList("Alice", "Bob", "Charlie");

Collections.sort(names, (a, b) -> a.compareTo(b));
```

Or even shorter:

```java
names.sort((a, b) -> a.compareTo(b));
```

The intent is immediately clear.

---

# What Is a Lambda Expression?

A lambda expression is an **anonymous function**.

Unlike a normal method, it:

- Has no name
- Has no access modifier
- Is not declared inside a class
- Can be passed as an argument
- Can be returned from methods

General syntax:

```java
(parameters) -> expression
```

or

```java
(parameters) -> {
    statements;
}
```

The arrow (`->`) is called the **lambda operator**.

---

# Lambda Syntax

General form:

```java
(parameters) -> body
```

Examples:

No parameters:

```java
() -> System.out.println("Hello");
```

One parameter:

```java
name -> System.out.println(name);
```

Multiple parameters:

```java
(a, b) -> a + b
```

Multiple statements:

```java
(a, b) -> {
    int sum = a + b;
    return sum;
}
```

---

# Understanding the Parts

Example:

```java
(x, y) -> x + y
```

Breaking it down:

| Part | Meaning |
|------|----------|
| `(x, y)` | Parameters |
| `->` | Lambda operator |
| `x + y` | Lambda body |

Equivalent method:

```java
public int add(int x, int y) {
    return x + y;
}
```

The compiler infers the parameter types from the target functional interface.

---

# Type Inference

Java automatically infers parameter types.

Instead of:

```java
(int a, int b) -> a + b
```

We usually write:

```java
(a, b) -> a + b
```

The compiler already knows the expected types.

Example:

```java
Comparator<String> comparator =
        (a, b) -> a.compareTo(b);
```

Here, the compiler knows that `a` and `b` are `String`.

---

# Expression Lambda vs Block Lambda

## Expression Lambda

Single expression.

```java
x -> x * x
```

The return value is implicit.

Equivalent:

```java
x -> {
    return x * x;
}
```

---

## Block Lambda

Multiple statements.

```java
x -> {
    System.out.println(x);
    return x * x;
}
```

Whenever braces are used, an explicit `return` is required for non-`void` methods.

---

# Lambda with No Parameters

```java
Runnable task = () -> System.out.println("Running");

task.run();
```

Output:

```text
Running
```

---

# Lambda with One Parameter

```java
Consumer<String> printer =
        name -> System.out.println(name);

printer.accept("Alice");
```

Output:

```text
Alice
```

Parentheses are optional when there is exactly one parameter and its type is inferred.

---

# Lambda with Multiple Parameters

```java
BinaryOperator<Integer> add =
        (a, b) -> a + b;

System.out.println(add.apply(10, 20));
```

Output:

```text
30
```

---

# Lambda Returning Boolean

```java
Predicate<Integer> isEven =
        number -> number % 2 == 0;

System.out.println(isEven.test(10));
```

Output:

```text
true
```

---

# Lambdas and Functional Interfaces

A lambda expression does **not** have its own type.

It always targets a **Functional Interface**.

Example:

```java
Runnable task =
        () -> System.out.println("Running");
```

Here:

- Lambda → implementation
- `Runnable` → target functional interface

Without a functional interface, a lambda cannot exist.

We'll explore functional interfaces in the next section.

---

# Variable Capture

A lambda can access variables from the enclosing scope.

Example:

```java
String prefix = "Hello";

Consumer<String> consumer =
        name -> System.out.println(prefix + " " + name);

consumer.accept("Alice");
```

Output:

```text
Hello Alice
```

---

# Effectively Final Variables

Captured local variables must be **final or effectively final**.

Valid:

```java
int x = 10;

Runnable r = () -> System.out.println(x);
```

Invalid:

```java
int x = 10;

x++;

Runnable r = () -> System.out.println(x);
```

Compilation error:

```text
Local variable x defined in an enclosing scope
must be final or effectively final
```

Reason:

The lambda captures the variable's value, not the variable itself.

---

# Accessing Instance Variables

Unlike local variables, instance variables may be modified.

```java
public class Counter {

    private int count = 0;

    public void increment() {
        Runnable task = () -> count++;
        task.run();
    }
}
```

This is perfectly valid.

---

# Accessing Static Variables

Static variables are also accessible.

```java
class Demo {

    static int counter = 0;

    public static void main(String[] args) {

        Runnable r = () -> counter++;

        r.run();

        System.out.println(counter);
    }
}
```

Output:

```text
1
```

---

# The `this` Keyword

Inside a lambda, `this` refers to the enclosing object.

Example:

```java
public class Demo {

    public void print() {

        Runnable r = () -> {
            System.out.println(this);
        };

        r.run();
    }
}
```

Unlike anonymous classes, lambdas do **not** create a new `this` context.

---

# Lambda vs Anonymous Class

| Lambda | Anonymous Class |
|----------|----------------|
| Less boilerplate | More verbose |
| No new anonymous type | Creates a new anonymous class |
| `this` refers to enclosing class | `this` refers to anonymous object |
| Better readability | More ceremony |
| Preferred for functional interfaces | Useful when multiple methods or state are required |

---

# Performance

A common misconception is that lambdas always create anonymous objects.

In reality, the JVM uses **`invokedynamic`** and **`LambdaMetafactory`** to create lambda implementations efficiently.

Compared to anonymous classes, lambdas generally:

- Generate fewer synthetic classes
- Reduce class-loading overhead
- Allow JVM optimizations
- May improve startup and runtime performance

We'll discuss `invokedynamic` in more detail later in this chapter.

---

# Best Practices

- Keep lambda bodies short.
- Prefer expression lambdas when possible.
- Avoid complex business logic inside lambdas.
- Use method references when they improve readability.
- Avoid modifying external mutable state inside lambdas.

---

# Interview Questions

### Q1. What is a lambda expression?

A lambda expression is an anonymous function that provides the implementation of a functional interface using concise syntax.

---

### Q2. Why were lambda expressions introduced?

To reduce boilerplate code, support functional programming, improve readability, and enable APIs such as the Stream API.

---

### Q3. Can a lambda exist without a functional interface?

No.

Every lambda expression must target a functional interface.

---

### Q4. Why must captured local variables be effectively final?

Because lambdas capture the value of local variables. Allowing those variables to change after capture would lead to inconsistent behavior and thread-safety issues.

---

### Q5. How is `this` interpreted inside a lambda?

Inside a lambda, `this` refers to the enclosing instance, unlike anonymous classes where `this` refers to the anonymous object itself.

---

# Key Takeaways

- Lambda expressions introduce functional programming capabilities to Java.
- They replace much of the boilerplate associated with anonymous classes.
- Every lambda targets a functional interface.
- Local variables captured by lambdas must be final or effectively final.
- Lambdas are implemented efficiently using `invokedynamic`, enabling better JVM optimizations.
- Clean, concise lambdas improve code readability and are fundamental to modern Java development.
