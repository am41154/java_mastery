# 7.11 Collection Factory Methods (`List.of()`, `Set.of()`, `Map.of()`)

Java 9 introduced **Collection Factory Methods**, making it much easier to create small, immutable collections.

Before Java 9, developers often wrote:

```java
List<String> list = new ArrayList<>();

list.add("Java");
list.add("Spring");
list.add("Kafka");
```

Or:

```java
List<String> list =
        Arrays.asList("Java", "Spring", "Kafka");
```

While functional, these approaches have drawbacks:

- More verbose
- `Arrays.asList()` creates a fixed-size list, not a truly immutable one
- Additional code is needed to create unmodifiable collections

Java 9 solved this by introducing convenient factory methods.

---

# Why Were Collection Factory Methods Introduced?

Goals:

- Reduce boilerplate
- Encourage immutability
- Improve readability
- Optimize memory for small collections
- Simplify constant collection creation

Example:

```java
List<String> technologies =
        List.of(
                "Java",
                "Spring",
                "Kafka"
        );
```

Much cleaner than previous approaches.

---

# List.of()

Creates an immutable list.

Example:

```java
List<String> languages =
        List.of(
                "Java",
                "Python",
                "Go"
        );
```

Output:

```text
[Java, Python, Go]
```

---

# Immutability

Attempting to modify the list results in an exception.

```java
languages.add("Rust");
```

Output:

```text
UnsupportedOperationException
```

Similarly:

```java
languages.remove("Java");

languages.set(0, "C++");
```

All modification operations fail.

---

# Set.of()

Creates an immutable set.

```java
Set<String> frameworks =
        Set.of(
                "Spring",
                "Hibernate",
                "Micronaut"
        );
```

---

# Duplicate Elements

Unlike `HashSet`, duplicates are not silently ignored.

Example:

```java
Set.of("Java", "Java");
```

Output:

```text
IllegalArgumentException

duplicate element
```

This helps detect programming errors early.

---

# Map.of()

Creates an immutable map.

Example:

```java
Map<Integer, String> employees =
        Map.of(
                1, "Alice",
                2, "Bob",
                3, "Charlie"
        );
```

Output:

```text
{1=Alice, 2=Bob, 3=Charlie}
```

---

# Duplicate Keys

```java
Map.of(
    1, "Alice",
    1, "Bob"
);
```

Output:

```text
IllegalArgumentException

duplicate key
```

---

# Map.ofEntries()

`Map.of()` supports up to **10 key-value pairs**.

For larger maps:

```java
Map<Integer, String> map =
        Map.ofEntries(
                Map.entry(1, "A"),
                Map.entry(2, "B"),
                Map.entry(3, "C"),
                Map.entry(4, "D")
        );
```

Suitable for creating larger immutable maps.

---

# Null Values Are Not Allowed

Unlike `HashMap` or `ArrayList`, factory methods reject `null`.

Example:

```java
List.of(
        "Java",
        null
);
```

Output:

```text
NullPointerException
```

Similarly:

```java
Set.of(null);
```

and

```java
Map.of(
    1,
    null
);
```

also throw `NullPointerException`.

---

# Arrays.asList() vs List.of()

This is a common interview topic.

Example:

```java
List<String> list =
        Arrays.asList("A", "B");
```

Can change existing elements:

```java
list.set(0, "X");
```

Works.

Cannot change size:

```java
list.add("C");
```

Throws:

```text
UnsupportedOperationException
```

Now compare with:

```java
List<String> list =
        List.of("A", "B");
```

Neither of these are allowed:

```java
list.set(0, "X");

list.add("C");
```

The collection is completely immutable.

---

# Comparison

| Feature | Arrays.asList() | List.of() |
|----------|-----------------|-----------|
| Add | ❌ | ❌ |
| Remove | ❌ | ❌ |
| Replace (`set`) | ✅ | ❌ |
| Allows null | ✅ | ❌ |
| Immutable | No (fixed-size) | Yes |

---

# Mutable Copy

Sometimes an immutable collection needs to be modified.

Example:

```java
List<String> immutable =
        List.of(
                "Java",
                "Spring"
        );
```

Create a mutable copy:

```java
List<String> mutable =
        new ArrayList<>(immutable);

mutable.add("Kafka");
```

Output:

```text
[Java, Spring, Kafka]
```

---

# copyOf()

Java 10 introduced `copyOf()` methods.

Example:

```java
List<String> list =
        List.copyOf(existingList);
```

Similarly:

```java
Set.copyOf(existingSet);

Map.copyOf(existingMap);
```

These create immutable copies of existing collections.

---

# Defensive Copying

Consider:

```java
public class Employee {

    private final List<String> skills;

    public Employee(List<String> skills) {
        this.skills = skills;
    }
}
```

Problem:

External code can still modify the list.

Better:

```java
public Employee(List<String> skills) {

    this.skills =
            List.copyOf(skills);
}
```

Now the object's internal state cannot be modified through the original reference.

This is known as **defensive copying**.

---

# Performance

Collection factory methods are optimized for small immutable collections.

Example:

```java
List.of(
        "A",
        "B",
        "C"
);
```

Internally, the JDK uses specialized implementations instead of general-purpose `ArrayList`.

Benefits include:

- Lower memory usage
- Faster creation
- Better cache locality
- No resizing overhead

---

# Internal Implementations

Internally, the JDK uses specialized classes such as:

```text
ImmutableCollections.List12

ImmutableCollections.ListN

ImmutableCollections.Set12

ImmutableCollections.MapN
```

These are implementation details and should not be relied upon in application code.

# Real-World Usage

Configuration values:

```java
private static final List<String> SUPPORTED_CURRENCIES =
        List.of(
                "USD",
                "EUR",
                "INR"
        );
```

REST API validation:

```java
private static final Set<String> ALLOWED_STATUS =
        Set.of(
                "ACTIVE",
                "INACTIVE",
                "PENDING"
        );
```

Error code mapping:

```java
Map<Integer, String> errors =
        Map.of(
                404, "Not Found",
                500, "Internal Error"
        );
```

These collections are naturally immutable and thread-safe.

---

# Best Practices

- Prefer `List.of()`, `Set.of()`, and `Map.of()` for constant collections.
- Use `copyOf()` when creating immutable defensive copies.
- Do not expect factory-created collections to allow `null`.
- Create mutable copies only when modification is genuinely required.
- Prefer immutable collections whenever possible to reduce bugs and improve thread safety.

---

# Common Mistakes

## Assuming `List.of()` Returns `ArrayList`

Incorrect assumption:

```java
ArrayList<String> list =
        (ArrayList<String>) List.of("A");
```

Results in:

```text
ClassCastException
```

The returned implementation is JDK-specific and should be treated as a `List`.

---

## Expecting Mutability

```java
List<String> list =
        List.of("A");

list.add("B");
```

Throws:

```text
UnsupportedOperationException
```

---

## Using null Values

```java
List.of(
        "Java",
        null
);
```

Throws:

```text
NullPointerException
```

# Interview Questions

### Q1. What is the difference between `Arrays.asList()` and `List.of()`?

`Arrays.asList()` returns a fixed-size list where elements can be replaced using `set()`, whereas `List.of()` returns a truly immutable list that does not allow any modifications.

---

### Q2. Are collections created using `List.of()` mutable?

No. They are immutable. Any attempt to add, remove, or replace elements results in an `UnsupportedOperationException`.

---

### Q3. Can `List.of()` contain `null` elements?

No. Passing `null` to `List.of()`, `Set.of()`, or `Map.of()` results in a `NullPointerException`.

---

### Q4. What is the purpose of `List.copyOf()`?

`List.copyOf()` creates an immutable copy of an existing collection and is commonly used for defensive copying.

---

### Q5. Why should immutable collections be preferred?

Immutable collections:

- Prevent accidental modification
- Simplify reasoning about code
- Improve thread safety
- Reduce synchronization requirements
- Encourage functional programming practices

---

# Key Takeaways

- Java 9 introduced convenient factory methods for creating immutable collections.
- `List.of()`, `Set.of()`, and `Map.of()` produce immutable collections that reject `null` values.
- `Map.ofEntries()` supports creating larger immutable maps.
- `copyOf()` methods provide a simple way to create immutable defensive copies.
- Immutable collections are widely used in modern Java applications for configuration, constants, and thread-safe shared data.
- Understanding the distinction between fixed-size and immutable collections is a frequent interview topic.
