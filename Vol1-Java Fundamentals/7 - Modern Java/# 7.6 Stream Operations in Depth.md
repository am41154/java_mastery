# map()

The `map()` operation transforms each element into another element.

General syntax:

```java
Stream<R> map(Function<T, R> mapper)
```

Example:

```java
List<String> names =
        List.of("alice", "bob", "charlie");

List<String> result =
        names.stream()
             .map(String::toUpperCase)
             .toList();
```

Output:

```text
[ALICE, BOB, CHARLIE]
```

Transformation:

```text
alice

↓

ALICE
```

One input produces exactly one output.

---

# When to Use map()

Use `map()` when every input element produces exactly one output element.

Examples:

- String → Uppercase String
- Employee → EmployeeDTO
- Integer → Square
- Product → Price

---

# flatMap()

Sometimes one input produces multiple outputs.

Example:

```java
List<List<String>> data =
        List.of(
                List.of("A", "B"),
                List.of("C", "D")
        );
```

Without `flatMap()`:

```java
data.stream()
    .map(List::stream)
```

Result:

```text
Stream<Stream<String>>
```

This is usually not what we want.

Instead:

```java
List<String> result =
        data.stream()
            .flatMap(List::stream)
            .toList();
```

Output:

```text
[A, B, C, D]
```

---

# map() vs flatMap()

Consider:

```java
List<String> words =
        List.of("Java", "Spring");
```

Using `map()`:

```java
words.stream()
     .map(word -> word.split(""))
```

Result:

```text
Stream<String[]>
```

Using `flatMap()`:

```java
words.stream()
     .map(word -> word.split(""))
     .flatMap(Arrays::stream)
```

Output:

```text
J
a
v
a
S
p
r
i
n
g
```

---

# Visual Difference

Using `map()`:

```text
A

↓

[A1 A2]
```

Produces:

```text
Stream<List>
```

Using `flatMap()`:

```text
A

↓

A1

A2
```

Produces:

```text
Single Flat Stream
```

# reduce()

`reduce()` combines stream elements into a single value.

General syntax:

```java
reduce(identity, accumulator)
```

Example:

```java
List<Integer> numbers =
        List.of(1,2,3,4,5);

int sum =
        numbers.stream()
               .reduce(0, Integer::sum);
```

Output:

```text
15
```

---

# How reduce() Works

```text
0

↓

0 + 1 = 1

↓

1 + 2 = 3

↓

3 + 3 = 6

↓

6 + 4 = 10

↓

10 + 5 = 15
```

---

# reduce() Without Identity

```java
Optional<Integer> max =
        numbers.stream()
               .reduce(Integer::max);
```

Output:

```java
Optional[5]
```

Because the stream may be empty, the result is wrapped in `Optional`.

---

# Common reduce() Examples

Find maximum:

```java
numbers.stream()
       .reduce(Integer::max);
```

Multiply:

```java
numbers.stream()
       .reduce(1, (a, b) -> a * b);
```

Concatenate:

```java
words.stream()
     .reduce("", String::concat);
```

# collect()

`collect()` converts a stream into another data structure.

Example:

```java
List<String> result =
        names.stream()
             .collect(Collectors.toList());
```

Since Java 16:

```java
List<String> result =
        names.stream()
             .toList();
```

is generally preferred when an unmodifiable list is sufficient.

---

# Common Collectors

The `Collectors` utility class provides many useful collectors.

Popular ones include:

- toList()
- toSet()
- joining()
- counting()
- groupingBy()
- partitioningBy()
- mapping()
- summarizingInt()
- averagingInt()
- collectingAndThen()

---

# toSet()

```java
Set<String> result =
        names.stream()
             .collect(Collectors.toSet());
```

Duplicates are removed.

---

# joining()

```java
String result =
        names.stream()
             .collect(Collectors.joining(", "));
```

Output:

```text
Alice, Bob, Charlie
```

---

# counting()

```java
long total =
        names.stream()
             .collect(Collectors.counting());
```

Equivalent:

```java
names.stream().count();
```

---

# groupingBy()

One of the most important collectors.

Example:

```java
List<String> names =
        List.of("Alice",
                "Adam",
                "Bob",
                "Brian");
```

Group by first letter:

```java
Map<Character, List<String>> result =
        names.stream()
             .collect(
                 Collectors.groupingBy(
                     name -> name.charAt(0)
                 )
             );
```

Output:

```text
A

Alice

Adam

B

Bob

Brian
```

---

# Another groupingBy() Example

Suppose:

```java
class Employee {

    String department;
}
```

Group employees:

```java
Map<String, List<Employee>> employees =
        employeeList.stream()
                    .collect(
                        Collectors.groupingBy(
                            Employee::getDepartment
                        )
                    );
```

Very common in enterprise applications.

---

# partitioningBy()

Unlike `groupingBy()`, `partitioningBy()` always creates **two groups**.

Example:

```java
Map<Boolean, List<Integer>> result =
        numbers.stream()
               .collect(
                   Collectors.partitioningBy(
                       n -> n % 2 == 0
                   )
               );
```

Output:

```text
true

2
4
6

false

1
3
5
```

---

# groupingBy() vs partitioningBy()

| groupingBy() | partitioningBy() |
|--------------|------------------|
| Multiple groups | Exactly two groups |
| Key can be any type | Key is always Boolean |
| More flexible | Optimized for true/false conditions |

---

# summarizingInt()

Useful for statistics.

```java
IntSummaryStatistics stats =
        numbers.stream()
               .collect(
                   Collectors.summarizingInt(
                       Integer::intValue
                   )
               );
```

Provides:

- Count
- Sum
- Average
- Minimum
- Maximum

All in a single pass.

---

# averagingInt()

```java
double average =
        numbers.stream()
               .collect(
                   Collectors.averagingInt(
                       Integer::intValue
                   )
               );
```

---

# mapping()

Allows additional transformation inside another collector.

Example:

```java
Map<String, List<String>> result =
        employees.stream()
                 .collect(
                     Collectors.groupingBy(
                         Employee::getDepartment,
                         Collectors.mapping(
                             Employee::getName,
                             Collectors.toList()
                         )
                     )
                 );
```

Produces:

```text
IT

John
Alice

HR

Bob
```

# Parallel Streams

Streams can execute in parallel.

```java
numbers.parallelStream()
       .map(this::calculate)
       .toList();
```

Internally:

```text
CPU Core 1

↓

CPU Core 2

↓

CPU Core 3

↓

Merge Results
```

Parallel Streams use the **ForkJoinPool.commonPool()** by default.

---

# When Parallel Streams Help

Suitable when:

- Large datasets
- CPU-intensive calculations
- Independent operations
- Multi-core machines

Example:

- Image processing
- Scientific calculations
- Financial analytics

---

# When Parallel Streams Hurt Performance

Avoid Parallel Streams when:

- Dataset is small
- Operations involve blocking I/O
- Order must be strictly preserved
- Shared mutable state exists

Thread coordination overhead may outweigh any performance gains.

---

# Stream Performance Tips

Prefer:

```java
IntStream
```

instead of:

```java
Stream<Integer>
```

because it avoids boxing.

Example:

```java
IntStream.range(1, 100)
         .sum();
```

is generally more efficient than using `Stream<Integer>`.

---

# Common Mistakes

## Modifying Source Collection

Incorrect:

```java
list.stream()
    .forEach(list::remove);
```

May throw:

```text
ConcurrentModificationException
```

---

## Side Effects

Avoid:

```java
List<String> result =
        new ArrayList<>();

names.stream()
     .forEach(result::add);
```

Prefer collecting directly:

```java
List<String> result =
        names.stream()
             .toList();
```

---

## Reusing Streams

Incorrect:

```java
Stream<String> stream =
        names.stream();

stream.count();

stream.toList();
```

Runtime exception:

```text
Stream has already been operated upon or closed
```

---

# Best Practices

- Prefer `map()` for one-to-one transformations.
- Use `flatMap()` to flatten nested structures.
- Use `reduce()` for aggregation.
- Use `collect()` for building collections and maps.
- Prefer `groupingBy()` for categorization.
- Use primitive streams (`IntStream`, `LongStream`, `DoubleStream`) when appropriate.
- Use Parallel Streams only after measuring performance.

---

# Interview Questions

### Q1. What is the difference between `map()` and `flatMap()`?

- `map()` transforms one input into one output.
- `flatMap()` transforms one input into zero, one, or many outputs and flattens the resulting streams into a single stream.

---

### Q2. What is the purpose of `reduce()`?

`reduce()` combines stream elements into a single result, such as a sum, maximum, concatenated string, or custom aggregate.

---

### Q3. What is the difference between `groupingBy()` and `partitioningBy()`?

- `groupingBy()` creates multiple groups based on a classifier.
- `partitioningBy()` always creates exactly two groups based on a boolean predicate.

---

### Q4. When should Parallel Streams be used?

Parallel Streams are beneficial for CPU-intensive processing of large datasets with independent operations. They are generally not suitable for blocking I/O or small datasets.

---

### Q5. Why are primitive streams (`IntStream`, `LongStream`, `DoubleStream`) preferred in performance-sensitive code?

They avoid boxing and unboxing overhead, resulting in lower memory usage and better performance.

---

# Key Takeaways

- `map()` performs one-to-one transformations, while `flatMap()` flattens nested structures.
- `reduce()` aggregates stream elements into a single result.
- `collect()` and `Collectors` provide powerful mechanisms for building collections, grouping, partitioning, and summarizing data.
- Parallel Streams can improve throughput for CPU-bound workloads but should be used only after performance evaluation.
- Choosing the right stream operation leads to cleaner, more expressive, and more efficient Java code.
