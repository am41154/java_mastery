# 7.7 Collectors API Deep Dive

The **Collectors API** is one of the most powerful components of the Stream API.

While `map()`, `filter()`, and `reduce()` transform data, **Collectors** define **how the final result should be accumulated**.

Most enterprise Stream code eventually ends with a collector.

Example:

```java
employees.stream()
         .filter(Employee::isActive)
         .collect(Collectors.toList());
```

In real-world applications, Collectors are heavily used for:

- Report generation
- Dashboard statistics
- Data aggregation
- Grouping records
- Building lookup maps
- Data transformation

---

# What is a Collector?

A **Collector** is an object that accumulates elements from a stream into a final result.

General syntax:

```java
stream.collect(Collector)
```

Pipeline:

```text
Stream

↓

Collector

↓

Final Result
```

The result can be:

- List
- Set
- Map
- String
- Statistics
- Custom Object

---

# Commonly Used Collectors

| Collector | Result |
|------------|---------|
| toList() | List |
| toSet() | Set |
| toMap() | Map |
| joining() | String |
| groupingBy() | Map<Group, List> |
| partitioningBy() | Map<Boolean, List> |
| counting() | Long |
| mapping() | Transformed values |
| filtering() | Filter within grouping |
| collectingAndThen() | Post-process result |
| teeing() | Combine two collectors |
| summarizingInt() | Statistics |

# toList()

Collects stream elements into a `List`.

```java
List<String> result =
        names.stream()
             .collect(Collectors.toList());
```

Java 16 introduced:

```java
List<String> result =
        names.stream()
             .toList();
```

Use `toList()` when you don't specifically need a mutable list.

---

# toSet()

Collects unique elements.

```java
Set<String> result =
        names.stream()
             .collect(Collectors.toSet());
```

Input:

```text
A
A
B
C
```

Output:

```text
A
B
C
```

---

# toMap()

Creates a map.

Example:

```java
List<Employee> employees = ...
```

```java
Map<Integer, Employee> map =
        employees.stream()
                 .collect(
                     Collectors.toMap(
                         Employee::getId,
                         Function.identity()
                     )
                 );
```

Result:

```text
Employee ID

↓

Employee Object
```

---

# Duplicate Keys

Suppose:

```text
ID

101

101
```

This causes:

```text
IllegalStateException

Duplicate Key
```

Solution:

```java
Collectors.toMap(
    Employee::getId,
    Function.identity(),
    (oldValue, newValue) -> oldValue
)
```

The third parameter specifies the merge strategy.

---

# joining()

Concatenates strings.

```java
String result =
        names.stream()
             .collect(Collectors.joining(", "));
```

Output:

```text
Alice, Bob, Charlie
```

With prefix and suffix:

```java
Collectors.joining(", ", "[", "]")
```

Output:

```text
[Alice, Bob, Charlie]
```

---

# counting()

```java
long total =
        employees.stream()
                 .collect(Collectors.counting());
```

Equivalent to:

```java
employees.stream().count();
```

Useful as a downstream collector.

# groupingBy()

One of the most important collectors.

Example:

```java
Map<String, List<Employee>> result =
        employees.stream()
                 .collect(
                     Collectors.groupingBy(
                         Employee::getDepartment
                     )
                 );
```

Output:

```text
IT

↓

Employees

HR

↓

Employees
```

---

# Multi-Level Grouping

Example:

```java
Map<String,
    Map<String, List<Employee>>> result =
employees.stream()
         .collect(
             Collectors.groupingBy(
                 Employee::getDepartment,
                 Collectors.groupingBy(
                     Employee::getCity
                 )
             )
         );
```

Result:

```text
Department

↓

City

↓

Employees
```

Very common in reporting applications.

---

# groupingBy() with counting()

Example:

```java
Map<String, Long> result =
        employees.stream()
                 .collect(
                     Collectors.groupingBy(
                         Employee::getDepartment,
                         Collectors.counting()
                     )
                 );
```

Output:

```text
IT → 10

HR → 4
```

---

# groupingBy() with mapping()

Suppose we only need employee names.

```java
Map<String, List<String>> result =
employees.stream()
         .collect(
             Collectors.groupingBy(
                 Employee::getDepartment,
                 Collectors.mapping(
                     Employee::getName,
                     Collectors.toList()
                 )
             )
         );
```

Output:

```text
IT

↓

John

Alice
```

---

# filtering() (Java 9)

Allows filtering inside a collector.

Example:

```java
Map<String, List<Employee>> result =
employees.stream()
         .collect(
             Collectors.groupingBy(
                 Employee::getDepartment,
                 Collectors.filtering(
                     Employee::isActive,
                     Collectors.toList()
                 )
             )
         );
```

Useful when combining grouping and filtering in a single collector.

---

# collectingAndThen()

Performs a finishing transformation.

Example:

```java
List<String> result =
names.stream()
     .collect(
         Collectors.collectingAndThen(
             Collectors.toList(),
             Collections::unmodifiableList
         )
     );
```

Process:

```text
Collect

↓

Transform Result

↓

Return Final Object
```

---

# partitioningBy()

Partitions into two groups.

```java
Map<Boolean, List<Employee>> result =
employees.stream()
         .collect(
             Collectors.partitioningBy(
                 Employee::isActive
             )
         );
```

Output:

```text
true

↓

Active Employees

false

↓

Inactive Employees
```

---

# summarizingInt()

Provides multiple statistics in one pass.

```java
IntSummaryStatistics stats =
employees.stream()
         .collect(
             Collectors.summarizingInt(
                 Employee::getSalary
             )
         );
```

Provides:

- Count
- Sum
- Average
- Min
- Max

Without multiple traversals.

---

# averagingInt()

```java
double avg =
employees.stream()
         .collect(
             Collectors.averagingInt(
                 Employee::getSalary
             )
         );
```

---

# maxBy() and minBy()

Highest salary:

```java
Optional<Employee> highest =
employees.stream()
         .collect(
             Collectors.maxBy(
                 Comparator.comparing(
                     Employee::getSalary
                 )
             )
         );
```

Lowest salary:

```java
Collectors.minBy(...)
```

---

# teeing() (Java 12)

One of the newer collectors.

Allows two collectors to execute simultaneously.

Example:

```java
record SalaryStats(
        Double average,
        Long count
) {}
```

```java
SalaryStats stats =
employees.stream()
         .collect(
             Collectors.teeing(
                 Collectors.averagingInt(Employee::getSalary),
                 Collectors.counting(),
                 SalaryStats::new
             )
         );
```

Result:

```text
Average Salary

+

Employee Count
```

Single traversal.

# Custom Collector

Although uncommon in everyday development, Java allows creating custom collectors using:

```java
Collector.of(...)
```

Components include:

- Supplier
- Accumulator
- Combiner
- Finisher
- Characteristics

This is useful for specialized aggregation logic but is generally reserved for advanced use cases.

---

# Performance Considerations

Collectors are optimized for:

- Sequential Streams
- Parallel Streams
- Immutable accumulation
- Efficient reductions

Avoid:

```java
stream.forEach(list::add);
```

Prefer:

```java
stream.collect(Collectors.toList());
```

The collector is designed for efficient accumulation and parallel execution.

---

# Best Practices

- Prefer built-in collectors over custom implementations.
- Use `groupingBy()` for categorization.
- Use `partitioningBy()` only for boolean conditions.
- Handle duplicate keys when using `toMap()`.
- Prefer downstream collectors (`mapping()`, `counting()`, `filtering()`) instead of multiple stream traversals.
- Use `teeing()` when multiple aggregate results can be computed in one pass.

---

# Interview Questions

### Q1. What is the purpose of the Collectors API?

The Collectors API defines how stream elements are accumulated into a final result such as a list, set, map, statistics, or custom object.

---

### Q2. What happens if duplicate keys are encountered in `Collectors.toMap()`?

By default, an `IllegalStateException` is thrown. A merge function must be supplied to resolve duplicate keys.

---

### Q3. What is the difference between `groupingBy()` and `partitioningBy()`?

- `groupingBy()` groups elements using any classifier.
- `partitioningBy()` always creates exactly two groups based on a boolean predicate.

---

### Q4. What is `collectingAndThen()` used for?

It performs a finishing transformation after the primary collector has completed, such as converting a mutable collection into an unmodifiable one.

---

### Q5. What is `teeing()`?

`teeing()` (introduced in Java 12) allows two collectors to process the same stream simultaneously and then combines their results into a single object.

---

# Key Takeaways

- Collectors define how stream elements are accumulated into final results.
- The `Collectors` utility class provides rich built-in collectors for lists, sets, maps, grouping, partitioning, statistics, and more.
- Downstream collectors enable powerful multi-level aggregation without additional stream traversals.
- `toMap()` requires careful handling of duplicate keys.
- Modern collectors like `filtering()` and `teeing()` simplify complex aggregation logic while improving readability and efficiency.
