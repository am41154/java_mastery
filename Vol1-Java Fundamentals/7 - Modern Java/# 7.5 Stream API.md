# 7.5 Stream API

The **Stream API**, introduced in **Java 8**, is one of the most significant additions to the Java language.

It enables developers to process collections of data using a **declarative**, **functional**, and often more readable approach.

Instead of focusing on **how** to process data, Stream API allows developers to focus on **what** needs to be done.

Today, Stream API is widely used in:

- Spring Boot applications
- Microservices
- Data processing pipelines
- REST APIs
- Batch processing
- Reactive programming foundations

---

# Why Was Stream API Introduced?

Before Java 8, processing collections usually involved loops.

Example:

```java
List<String> names = Arrays.asList("Alice", "Bob", "Charlie");

List<String> result = new ArrayList<>();

for (String name : names) {
    if (name.startsWith("A")) {
        result.add(name.toUpperCase());
    }
}
```

Although correct, this approach is:

- Verbose
- Imperative
- Difficult to compose
- Harder to parallelize

The Stream API simplifies the same operation.

```java
List<String> result = names.stream()
                           .filter(name -> name.startsWith("A"))
                           .map(String::toUpperCase)
                           .toList();
```

The intent is immediately obvious.

---

# What Is a Stream?

A **Stream** is a sequence of elements that supports various operations to produce a desired result.

A stream **does not store data**.

Instead, it processes data from a source.

Example:

```text
Collection

↓

Stream

↓

Operations

↓

Result
```

Common data sources include:

- Collections
- Arrays
- Files
- Maps
- Random values
- Generated sequences

---

# Stream vs Collection

Many developers confuse Collections and Streams.

| Collection | Stream |
|------------|---------|
| Stores data | Processes data |
| Mutable | Immutable pipeline |
| Can be traversed multiple times | Can be consumed only once |
| Eager | Lazy (most operations) |

Think of it this way:

- **Collection = Data**
- **Stream = Processing Pipeline**

---

# Stream Pipeline

A Stream pipeline consists of three stages.

```text
Source

↓

Intermediate Operations

↓

Terminal Operation
```

Example:

```java
names.stream()
     .filter(name -> name.startsWith("A"))
     .map(String::toUpperCase)
     .toList();
```

Breaking it down:

```
Source

↓

filter()

↓

map()

↓

toList()
```

# Creating Streams

## From Collection

```java
List<String> names =
        List.of("Alice", "Bob", "Charlie");

Stream<String> stream = names.stream();
```

---

## From Array

```java
String[] array = {"A", "B", "C"};

Stream<String> stream =
        Arrays.stream(array);
```

---

## Using Stream.of()

```java
Stream<String> stream =
        Stream.of("Java", "Spring", "Kafka");
```

---

## Empty Stream

```java
Stream<String> stream =
        Stream.empty();
```

---

## Infinite Stream

```java
Stream<Integer> stream =
        Stream.iterate(1, n -> n + 1);
```

Another example:

```java
Stream<Double> random =
        Stream.generate(Math::random);
```

These streams are infinite and usually combined with `limit()`.

---

# Intermediate Operations

Intermediate operations transform a stream and return another stream.

Characteristics:

- Lazy
- Chainable
- No processing occurs until a terminal operation is invoked

Examples:

- filter()
- map()
- flatMap()
- distinct()
- sorted()
- peek()
- limit()
- skip()

---

# filter()

Selects elements matching a condition.

```java
List<String> result =
        names.stream()
             .filter(name -> name.startsWith("A"))
             .toList();
```

Output:

```text
[Alice]
```

---

# map()

Transforms each element.

```java
List<String> result =
        names.stream()
             .map(String::toUpperCase)
             .toList();
```

Output:

```text
[ALICE, BOB, CHARLIE]
```

---

# filter() + map()

```java
List<String> result =
        names.stream()
             .filter(name -> name.length() > 3)
             .map(String::toUpperCase)
             .toList();
```

Processing pipeline:

```text
Collection

↓

filter()

↓

map()

↓

Result
```

---

# distinct()

Removes duplicates.

```java
List<Integer> numbers =
        List.of(1, 2, 2, 3, 1);

numbers.stream()
       .distinct()
       .toList();
```

Output:

```text
[1, 2, 3]
```

---

# sorted()

Sorts elements.

```java
names.stream()
     .sorted()
     .toList();
```

Reverse order:

```java
names.stream()
     .sorted(Comparator.reverseOrder())
     .toList();
```

---

# limit()

Restricts the number of elements.

```java
Stream.iterate(1, n -> n + 1)
      .limit(5)
      .toList();
```

Output:

```text
[1, 2, 3, 4, 5]
```

---

# skip()

Skips initial elements.

```java
List<Integer> result =
        List.of(1,2,3,4,5)
            .stream()
            .skip(2)
            .toList();
```

Output:

```text
[3,4,5]
```

---

# peek()

Useful for debugging.

```java
names.stream()
     .peek(System.out::println)
     .map(String::toUpperCase)
     .toList();
```

`peek()` should primarily be used for debugging or logging, not for modifying application state.

---

# Terminal Operations

Terminal operations end the pipeline.

Examples:

- forEach()
- collect()
- toList()
- reduce()
- count()
- anyMatch()
- allMatch()
- noneMatch()
- findFirst()
- findAny()
- min()
- max()

Once a terminal operation executes, the stream cannot be reused.

---

# forEach()

```java
names.stream()
     .forEach(System.out::println);
```

---

# count()

```java
long count =
        names.stream().count();
```

---

# findFirst()

```java
Optional<String> first =
        names.stream()
             .findFirst();
```

---

# anyMatch()

```java
boolean exists =
        names.stream()
             .anyMatch(name -> name.startsWith("A"));
```

---

# allMatch()

```java
boolean result =
        numbers.stream()
               .allMatch(n -> n > 0);
```

---

# noneMatch()

```java
boolean result =
        names.stream()
             .noneMatch(String::isEmpty);
```

---

# min() and max()

```java
Optional<Integer> min =
        numbers.stream()
               .min(Integer::compareTo);

Optional<Integer> max =
        numbers.stream()
               .max(Integer::compareTo);
```

# Lazy Evaluation

One of the most important Stream concepts.

Example:

```java
names.stream()
     .filter(name -> {
         System.out.println(name);
         return true;
     });
```

Output:

```text
Nothing
```

No terminal operation exists.

Therefore, **nothing executes**.

Now:

```java
names.stream()
     .filter(name -> {
         System.out.println(name);
         return true;
     })
     .count();
```

Output:

```text
Alice
Bob
Charlie
```

The pipeline executes only after the terminal operation.

---

# Streams Cannot Be Reused

Example:

```java
Stream<String> stream =
        names.stream();

stream.count();

stream.forEach(System.out::println);
```

Runtime exception:

```text
IllegalStateException

Stream has already been operated upon or closed
```

A stream is a one-time-use pipeline.

---

# Internal Iteration

Traditional loop:

```java
for (String name : names) {
    System.out.println(name);
}
```

The developer controls iteration.

With Streams:

```java
names.stream()
     .forEach(System.out::println);
```

The Stream API manages iteration internally.

This abstraction enables optimizations such as parallel execution.

---

# Best Practices

- Prefer Streams for data transformation and filtering.
- Avoid using Streams for complex stateful logic.
- Keep stream pipelines readable.
- Avoid side effects inside intermediate operations.
- Use method references where appropriate.
- Prefer `toList()` (Java 16+) over `collect(Collectors.toList())` when no mutable list is required.

# Interview Questions

### Q1. What is a Stream?

A Stream is a sequence of elements that supports functional-style operations for processing data from a source without storing it.

---

### Q2. What is the difference between a Collection and a Stream?

A Collection stores data, while a Stream processes data. Collections can be traversed multiple times, whereas a Stream can be consumed only once.

---

### Q3. What are Intermediate Operations?

Intermediate operations transform a stream and return another stream. They are lazy and do not execute until a terminal operation is invoked.

---

### Q4. What are Terminal Operations?

Terminal operations produce the final result or side effect, triggering execution of the entire stream pipeline.

---

### Q5. Why are Streams called lazy?

Because intermediate operations are not executed immediately. They are evaluated only when a terminal operation is performed.

---

# Key Takeaways

- Stream API enables declarative and functional data processing.
- A Stream pipeline consists of a source, intermediate operations, and a terminal operation.
- Intermediate operations are lazy and chainable.
- Terminal operations trigger pipeline execution.
- Streams do not store data and cannot be reused after a terminal operation.
- Stream API forms the foundation for many modern Java programming patterns.
