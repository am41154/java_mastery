# 7.3 Functional Interfaces

Lambda expressions cannot exist on their own.

Every lambda expression in Java is associated with a **Functional Interface**.

Functional Interfaces are the foundation of Java's functional programming model and are extensively used throughout the JDK and frameworks like Spring.

Understanding Functional Interfaces is essential for mastering:

- Lambda Expressions
- Method References
- Stream API
- CompletableFuture
- Reactive Programming

---

# What Is a Functional Interface?

A **Functional Interface** is an interface that contains **exactly one abstract method**.

Example:

```java
@FunctionalInterface
public interface Calculator {

    int calculate(int a, int b);
}
```

Since there is only one abstract method, this interface can be implemented using a lambda expression.

```java
Calculator addition = (a, b) -> a + b;

System.out.println(addition.calculate(10, 20));
```

Output:

```text
30
```

---

# Why Functional Interfaces?

Before Java 8, passing behavior required creating anonymous classes.

Example:

```java
Runnable task = new Runnable() {

    @Override
    public void run() {
        System.out.println("Running...");
    }
};
```

Java 8 introduced Functional Interfaces to simplify this.

```java
Runnable task = () -> System.out.println("Running...");
```

The compiler automatically understands that the lambda implements the single abstract method of `Runnable`.

---

# The @FunctionalInterface Annotation

Java provides the `@FunctionalInterface` annotation.

Example:

```java
@FunctionalInterface
public interface Printer {

    void print(String message);
}
```

This annotation is **optional**, but highly recommended.

Benefits:

- Improves readability.
- Documents developer intent.
- Compiler validates the interface.

---

# Compiler Validation

Suppose another developer accidentally adds a second abstract method.

```java
@FunctionalInterface
public interface Printer {

    void print();

    void display();
}
```

Compilation fails.

```text
Unexpected @FunctionalInterface annotation

Multiple non-overriding abstract methods found.
```

Without the annotation, the compiler would allow the interface, but it would no longer be usable with lambdas.

---

# Functional Interface Rules

A Functional Interface:

✔ Must contain exactly **one abstract method**.

It **may also contain**:

- Default methods
- Static methods
- Private methods (Java 9+)
- Methods inherited from `Object`

---

# Example with Default Method

```java
@FunctionalInterface
interface Printer {

    void print(String msg);

    default void start() {
        System.out.println("Starting...");
    }
}
```

Still valid.

Only one abstract method exists.

---

# Example with Static Method

```java
@FunctionalInterface
interface Printer {

    void print(String msg);

    static void version() {
        System.out.println("1.0");
    }
}
```

Still a Functional Interface.

---

# Object Methods Do Not Count

Example:

```java
@FunctionalInterface
interface Demo {

    void execute();

    boolean equals(Object obj);
}
```

Still valid.

Methods inherited from `java.lang.Object` are ignored when counting abstract methods.

---

# Invalid Functional Interface

```java
interface Demo {

    void execute();

    void stop();
}
```

This interface has:

```text
2 Abstract Methods
```

Therefore:

```text
Not a Functional Interface
```

---

# Functional Interface and Lambda

Consider:

```java
@FunctionalInterface
interface Greeting {

    void sayHello(String name);
}
```

Lambda implementation:

```java
Greeting greeting =
        name -> System.out.println("Hello " + name);

greeting.sayHello("Alice");
```

Output:

```text
Hello Alice
```

# Built-in Functional Interfaces

The JDK already provides many reusable Functional Interfaces.

The most commonly used ones are in:

```text
java.util.function
```

Instead of creating custom interfaces, prefer these standard interfaces whenever they fit your use case.

---

# Supplier<T>

Represents a producer of values.

Method:

```java
T get();
```

Example:

```java
Supplier<String> supplier =
        () -> "Hello";

System.out.println(supplier.get());
```

Output:

```text
Hello
```

Memory aid:

```
Supplier

Supplies Data
```

---

# Consumer<T>

Consumes a value but returns nothing.

Method:

```java
void accept(T value);
```

Example:

```java
Consumer<String> printer =
        name -> System.out.println(name);

printer.accept("Java");
```

Output:

```text
Java
```

Memory aid:

```
Consumer

Consumes Data
```

---

# Predicate<T>

Evaluates a condition.

Method:

```java
boolean test(T value);
```

Example:

```java
Predicate<Integer> even =
        n -> n % 2 == 0;

System.out.println(even.test(10));
```

Output:

```text
true
```

Memory aid:

```
Predicate

Returns True or False
```

---

# Function<T, R>

Transforms one value into another.

Method:

```java
R apply(T value);
```

Example:

```java
Function<String, Integer> length =
        str -> str.length();

System.out.println(length.apply("Java"));
```

Output:

```text
4
```

Memory aid:

```
Input

↓

Transformation

↓

Output
```

---

# UnaryOperator<T>

A special type of `Function`.

Input and output types are the same.

```java
UnaryOperator<Integer> square =
        x -> x * x;

System.out.println(square.apply(5));
```

Output:

```text
25
```

Equivalent to:

```java
Function<Integer, Integer>
```

---

# BinaryOperator<T>

Operates on two inputs of the same type and returns the same type.

```java
BinaryOperator<Integer> add =
        (a, b) -> a + b;

System.out.println(add.apply(5, 10));
```

Output:

```text
15
```

Equivalent to:

```java
BiFunction<T, T, T>
```

---

# BiFunction<T, U, R>

Accepts two input parameters.

```java
BiFunction<Integer, Integer, Integer> multiply =
        (a, b) -> a * b;

System.out.println(multiply.apply(4, 5));
```

Output:

```text
20
```

---

# BiPredicate<T, U>

Returns a boolean.

```java
BiPredicate<String, String> equals =
        (a, b) -> a.equals(b);

System.out.println(equals.test("Java", "Java"));
```

Output:

```text
true
```

---

# BiConsumer<T, U>

Consumes two values.

```java
BiConsumer<String, Integer> printer =
        (name, age) ->
                System.out.println(name + " " + age);

printer.accept("Alice", 25);
```

Output:

```text
Alice 25
```

---

# Primitive Functional Interfaces

Using generic types with primitives causes **boxing** and **unboxing**.

Example:

```java
Function<Integer, Integer>
```

Every call involves converting between:

```text
int

↓

Integer

↓

int
```

To avoid this overhead, Java provides primitive specializations.

Examples:

| Interface | Primitive |
|------------|-----------|
| IntFunction<R> | int |
| IntConsumer | int |
| IntSupplier | int |
| IntPredicate | int |
| LongSupplier | long |
| DoubleConsumer | double |

These improve performance in performance-critical code.

---

# Chaining Functional Interfaces

Many built-in interfaces provide composition methods.

Example:

```java
Function<String, String> trim =
        String::trim;

Function<String, String> upper =
        String::toUpperCase;

Function<String, String> result =
        trim.andThen(upper);

System.out.println(result.apply(" java "));
```

Output:

```text
JAVA
```

---

# Where Functional Interfaces Are Used

They appear throughout modern Java.

Examples:

- Stream API
- Collections sorting
- CompletableFuture
- Optional
- Executor Framework
- Spring Framework callbacks
- Event listeners

Understanding these interfaces makes many APIs much easier to use.

---

# Best Practices

- Prefer built-in functional interfaces over creating custom ones.
- Use `@FunctionalInterface` for custom interfaces.
- Choose the most appropriate interface (`Supplier`, `Consumer`, `Function`, `Predicate`, etc.).
- Use primitive specializations in performance-sensitive code.
- Keep lambda implementations simple and focused.

# Interview Questions

### Q1. What is a Functional Interface?

A Functional Interface is an interface containing exactly one abstract method, making it suitable as the target type for a lambda expression or method reference.

---

### Q2. Is `@FunctionalInterface` mandatory?

No.

It is optional but recommended because it allows the compiler to verify that the interface remains a valid functional interface.

---

### Q3. Can a Functional Interface contain default and static methods?

Yes.

Only abstract methods count toward the "single abstract method" rule. Default, static, private, and `Object` methods do not.

---

### Q4. What is the difference between `Function` and `Consumer`?

- `Function<T, R>` accepts a value and returns another value.
- `Consumer<T>` accepts a value but returns nothing.

---

### Q5. Why does Java provide primitive functional interfaces?

To avoid boxing and unboxing overhead when working with primitive types, improving performance.

---

# Key Takeaways

- Every lambda expression targets a Functional Interface.
- A Functional Interface contains exactly one abstract method.
- The `@FunctionalInterface` annotation helps enforce correctness.
- The `java.util.function` package provides a rich set of reusable functional interfaces.
- Primitive specializations improve performance by avoiding boxing.
- Functional Interfaces are fundamental to Streams, CompletableFuture, Optional, and many modern Java APIs.
