# 7.16 Switch Expressions (Java 14)

One of the oldest language constructs in Java is the `switch` statement.

For many years, it suffered from several limitations:

- Accidental fall-through
- Excessive boilerplate
- Temporary variables
- Poor readability
- Error-prone control flow

Java 14 introduced **Switch Expressions**, transforming `switch` from a statement into a true expression that can return values.

This modern syntax is:

- More concise
- Safer
- More readable
- Better suited for functional programming
- The foundation for Pattern Matching in Java 21

---

# Traditional Switch Statement

Example:

```java
String day = "MONDAY";

int workingHours;

switch (day) {

    case "MONDAY":
        workingHours = 8;
        break;

    case "SATURDAY":
        workingHours = 4;
        break;

    default:
        workingHours = 0;
}
```

Problems:

- Requires a temporary variable
- Requires `break`
- Easy to forget `break`
- More verbose

---

# The Fall-Through Problem

Consider:

```java
switch (day) {

    case "MONDAY":
        System.out.println("Monday");

    case "TUESDAY":
        System.out.println("Tuesday");
}
```

Output:

```text
Monday

Tuesday
```

Because `break` was omitted, execution falls through to the next case.

This has been a common source of bugs in Java applications.

---

# Switch Expression

Modern syntax:

```java
int hours = switch (day) {

    case "MONDAY" -> 8;

    case "SATURDAY" -> 4;

    default -> 0;
};
```

Notice:

- Returns a value
- No temporary variable
- No `break`
- No fall-through

---

# Arrow Syntax (`->`)

Traditional:

```java
case "A":

    ...

    break;
```

Modern:

```java
case "A" ->

    ...
```

The arrow guarantees that execution does not continue into another case.

---

# Multiple Labels

Instead of:

```java
case "MONDAY":

case "TUESDAY":

case "WEDNESDAY":
```

Modern syntax:

```java
case "MONDAY",
     "TUESDAY",
     "WEDNESDAY" -> 8;
```

Cleaner and easier to maintain.

---

# Returning Values

Example:

```java
String grade = "A";

String message = switch (grade) {

    case "A" -> "Excellent";

    case "B" -> "Good";

    case "C" -> "Average";

    default -> "Unknown";
};
```

Output:

```text
Excellent
```

---

# Block Cases

Sometimes multiple statements are required.

Example:

```java
int result = switch (day) {

    case "MONDAY" -> {

        System.out.println("Working Day");

        yield 8;
    }

    default -> 0;
};
```

---

# The `yield` Keyword

When a case contains a block, use `yield` to return the result.

Example:

```java
case "MONDAY" -> {

    log.info("Calculating");

    yield 8;
}
```

`yield` replaces assigning to temporary variables.

---

# Statement vs Expression

Traditional:

```java
switch (...) {

}
```

Performs an action.

Modern:

```java
int value = switch (...) {

};
```

Produces a value.

---

# Exhaustiveness

Switch expressions must always produce a value.

Therefore:

- Every possible case must be covered
- Or a `default` branch must exist

Example:

```java
switch (grade) {

    case "A" -> ...

}
```

Compilation error if some possibilities are not handled.

---

# Enum Example

Suppose:

```java
enum Status {

    NEW,

    PROCESSING,

    COMPLETED
}
```

Modern switch:

```java
String message =
switch (status) {

    case NEW ->

            "Created";

    case PROCESSING ->

            "Running";

    case COMPLETED ->

            "Finished";
};
```

No `default` is required because all enum constants are covered.

The compiler verifies exhaustiveness.

---

# Combining with Sealed Classes

Suppose:

```java
sealed interface Payment
        permits CardPayment,
                CashPayment {
}
```

Switch:

```java
return switch (payment) {

    case CardPayment card ->

            processCard(card);

    case CashPayment cash ->

            processCash(cash);
};
```

The compiler knows every possible implementation.

---

# Combining with Pattern Matching

Java 21:

```java
return switch (shape) {

    case Circle c ->

            calculateCircle(c);

    case Rectangle r ->

            calculateRectangle(r);
};
```

This is one of the primary motivations behind Switch Expressions.

---

# Nested Switch Expressions

Example:

```java
String priority =
switch (status) {

    case NEW ->

        switch(level){

            case 1 -> "LOW";

            default -> "HIGH";
        };

    default -> "NONE";
};
```

Since `switch` is now an expression, nesting becomes natural.

# Performance

Switch Expressions do not significantly change runtime performance.

Benefits include:

- Cleaner bytecode
- Better compiler optimization opportunities
- Reduced accidental fall-through
- Easier maintenance

Performance is generally equivalent to traditional switch statements.

---

# Best Practices

- Prefer Switch Expressions when computing a value.
- Use arrow (`->`) syntax instead of colon (`:`) syntax for new code.
- Use block cases with `yield` only when multiple statements are required.
- Avoid unnecessary nested switches.
- Prefer enums and sealed hierarchies with switch expressions for exhaustive handling.

---

# Common Mistakes

## Forgetting `yield`

Incorrect:

```java
case "A" -> {

    System.out.println("A");

    10;
}
```

Compilation error.

Correct:

```java
case "A" -> {

    System.out.println("A");

    yield 10;
}
```

---

## Mixing Arrow and Colon Syntax

Although legal in some contexts, mixing styles within the same switch reduces readability.

Prefer the modern arrow syntax consistently in new code.

---

## Using Switch Expressions for Complex Business Logic

A switch expression should remain focused on selecting a result.

If extensive logic is required, delegate to well-named methods.

---

# Switch Statement vs Switch Expression

| Feature | Switch Statement | Switch Expression |
|----------|-----------------|-------------------|
| Returns value | ❌ | ✅ |
| Requires `break` | Usually | ❌ |
| Fall-through | Yes | No |
| Uses `yield` | ❌ | Yes (block cases) |
| Temporary variable | Often | Not required |
| Readability | Moderate | High |

# Interview Questions

### Q1. What is the difference between a Switch Statement and a Switch Expression?

A Switch Statement performs actions, whereas a Switch Expression evaluates to a value that can be assigned, returned, or passed to another method.

---

### Q2. Why were Switch Expressions introduced?

To eliminate accidental fall-through, reduce boilerplate, improve readability, and allow `switch` to be used as an expression.

---

### Q3. What is the purpose of the `yield` keyword?

`yield` returns a value from a block case within a Switch Expression.

---

### Q4. Can Switch Expressions fall through?

No.

Arrow (`->`) cases do not support fall-through, making them safer than traditional switch statements.

---

### Q5. Why do Switch Expressions work well with Enums and Sealed Classes?

Because the compiler knows all possible values or permitted subclasses, it can verify that every case is handled, enabling exhaustive switches without requiring a `default` branch.

# Modern Java Example

```java
public sealed interface Payment
        permits CardPayment,
                CashPayment,
                UpiPayment {
}
```

```java
public record CardPayment(
        double amount
) implements Payment {
}
```

```java
public record CashPayment(
        double amount
) implements Payment {
}
```

```java
public record UpiPayment(
        double amount
) implements Payment {
}
```

Processing:

```java
public String process(Payment payment) {

    return switch (payment) {

        case CardPayment card ->
                "Card";

        case CashPayment cash ->
                "Cash";

        case UpiPayment upi ->
                "UPI";
    };
}
```

This combines:

- Records
- Sealed Interfaces
- Pattern Matching
- Switch Expressions

This style is becoming the standard for modern Java 21 applications.

# Key Takeaways

- Switch Expressions modernize one of Java's oldest language constructs.
- The arrow (`->`) syntax eliminates accidental fall-through and improves readability.
- `yield` allows block cases to return values.
- Switch Expressions integrate seamlessly with Enums, Records, Sealed Classes, and Pattern Matching.
- They encourage concise, expressive, and type-safe code while reducing boilerplate.
- Understanding Switch Expressions is essential for writing modern Java 17/21 code and is a common topic in senior backend interviews.s
