# 7.9 New Date and Time API (java.time)

One of the biggest improvements introduced in **Java 8** was the new **Date and Time API** (`java.time` package).

Before Java 8, developers primarily used:

- `java.util.Date`
- `java.util.Calendar`
- `SimpleDateFormat`

These classes had several design problems that frequently caused bugs in enterprise applications.

The Java 8 Date and Time API provides:

- Immutable classes
- Thread safety
- Better readability
- Rich API for date and time calculations
- Support for time zones
- ISO-8601 compliance

Today, `java.time` is the standard date/time API used in modern Java applications.

---

# Problems with the Old Date API

## 1. Mutable Objects

```java
Date date = new Date();

date.setYear(125);
```

The object itself changes.

This makes sharing instances across threads unsafe.

---

## 2. Not Thread Safe

Example:

```java
SimpleDateFormat sdf =
        new SimpleDateFormat("yyyy-MM-dd");
```

Using the same instance across multiple threads can produce incorrect results or runtime exceptions.

---

## 3. Poor API Design

Example:

```java
Date date = new Date();

System.out.println(date.getMonth());
```

Output:

```text
0 = January
```

Months start from **0**.

Similarly:

```java
date.getYear();
```

Returns:

```text
Current Year - 1900
```

These APIs are confusing and error-prone.

---

# The java.time Package

The Java 8 API introduces several specialized classes.

| Class | Purpose |
|---------|----------|
| LocalDate | Date only |
| LocalTime | Time only |
| LocalDateTime | Date + Time |
| Instant | Machine timestamp |
| ZonedDateTime | Date + Time + Time Zone |
| OffsetDateTime | Date + Time + UTC Offset |
| Duration | Time-based amount |
| Period | Date-based amount |
| DateTimeFormatter | Formatting and parsing |

---

# LocalDate

Represents a date without time.

Example:

```java
LocalDate today =
        LocalDate.now();

System.out.println(today);
```

Output:

```text
2026-07-12
```

---

# Creating LocalDate

Specific date:

```java
LocalDate birthday =
        LocalDate.of(1995, 5, 20);
```

From text:

```java
LocalDate date =
        LocalDate.parse("2026-12-25");
```

# LocalTime

Represents only time.

```java
LocalTime time =
        LocalTime.now();

System.out.println(time);
```

Example output:

```text
15:45:30.123
```

Specific time:

```java
LocalTime.of(10, 30);
```

# LocalDateTime

Represents date and time without timezone.

```java
LocalDateTime now =
        LocalDateTime.now();
```

Output:

```text
2026-07-12T15:45:30
```

Specific value:

```java
LocalDateTime.of(
        2026,
        7,
        12,
        15,
        30
);
```

---

# Instant

Represents a timestamp on the UTC timeline.

```java
Instant instant =
        Instant.now();
```

Output:

```text
2026-07-12T10:15:30Z
```

Useful for:

- Logging
- Distributed systems
- Databases
- Event timestamps

---

# ZonedDateTime

Represents a date/time with a timezone.

```java
ZonedDateTime now =
        ZonedDateTime.now();
```

Specific timezone:

```java
ZonedDateTime india =
        ZonedDateTime.now(
            ZoneId.of("Asia/Kolkata")
        );
```

Output:

```text
2026-07-12T15:45:30+05:30[Asia/Kolkata]
```

---

# OffsetDateTime

Contains date, time, and UTC offset.

```java
OffsetDateTime.now();
```

Example:

```text
2026-07-12T15:45:30+05:30
```

Unlike `ZonedDateTime`, it does not retain timezone rules such as daylight saving transitions.

---

# Immutability

All classes in `java.time` are immutable.

Example:

```java
LocalDate today =
        LocalDate.now();

LocalDate tomorrow =
        today.plusDays(1);
```

```
today

↓

2026-07-12

tomorrow

↓

2026-07-13
```

The original object remains unchanged.

Subtract:

# Period

Represents a date-based amount.

```java
LocalDate birth =
        LocalDate.of(1995,5,20);

Period age =
        Period.between(
            birth,
            LocalDate.now()
        );
```

Output:

```java
age.getYears();
```

Returns:

```text
31
```

---

# Duration

Represents a time-based amount.

Example:

```java
LocalTime start =
        LocalTime.of(10,0);

LocalTime end =
        LocalTime.of(12,30);

Duration duration =
        Duration.between(start,end);
```

Output:

```java
duration.toMinutes();
```

Returns:

```text
150
```

# Formatting Dates

Using `DateTimeFormatter`.

Example:

```java
LocalDate today =
        LocalDate.now();

String formatted =
today.format(
    DateTimeFormatter.ofPattern(
        "dd-MM-yyyy"
    )
);
```

Output:

```text
12-07-2026
```

---

# Parsing Dates

```java
String text =
        "25-12-2026";

LocalDate date =
LocalDate.parse(
    text,
    DateTimeFormatter.ofPattern(
        "dd-MM-yyyy"
    )
);
```

# Converting Between Time Zones

```java
ZonedDateTime india =
        ZonedDateTime.now(
                ZoneId.of("Asia/Kolkata")
        );

ZonedDateTime london =
india.withZoneSameInstant(
        ZoneId.of("Europe/London")
);
```

The instant remains the same while the displayed local time changes according to the target timezone.

# Best Practices

- Prefer `java.time` over `Date` and `Calendar`.
- Use `Instant` for timestamps stored in databases or exchanged between systems.
- Use `LocalDate` for birthdays, holidays, and dates without time.
- Use `LocalDateTime` when timezone information is unnecessary.
- Use `ZonedDateTime` when timezone rules matter.
- Always use `DateTimeFormatter` instead of `SimpleDateFormat`.

---

# Common Mistakes

## Using LocalDateTime for Global Timestamps

Incorrect:

```java
LocalDateTime.now();
```

for distributed systems.

Prefer:

```java
Instant.now();
```

or

```java
OffsetDateTime.now();
```

---

## Using System Default Time Zone Everywhere

Example:

```java
LocalDateTime.now();
```

This depends on the JVM's default timezone.

For distributed applications, explicitly specify the timezone when required.

---

## Using SimpleDateFormat

Avoid:

```java
SimpleDateFormat
```

It is mutable and not thread-safe.

Use:

```java
DateTimeFormatter
```

instead.

# Interview Questions

### Q1. Why was the `java.time` API introduced?

To replace the old `Date` and `Calendar` APIs with immutable, thread-safe, and easier-to-use date/time classes.

---

### Q2. What is the difference between `LocalDate` and `LocalDateTime`?

- `LocalDate` contains only a date.
- `LocalDateTime` contains both date and time but no timezone information.

---

### Q3. What is the difference between `Instant` and `LocalDateTime`?

- `Instant` represents a specific moment on the UTC timeline.
- `LocalDateTime` represents a local date and time without any timezone or offset.

---

### Q4. When should `ZonedDateTime` be used?

When timezone rules (including daylight saving time) are important, such as scheduling events across different geographic regions.

---

### Q5. Why is `DateTimeFormatter` preferred over `SimpleDateFormat`?

Because `DateTimeFormatter` is immutable, thread-safe, and integrates naturally with the `java.time` API.

---

# Key Takeaways

- The `java.time` API replaces the outdated `Date` and `Calendar` classes with a modern, immutable design.
- Choose the appropriate class (`LocalDate`, `Instant`, `ZonedDateTime`, etc.) based on the problem domain.
- Use `Period` for date-based calculations and `Duration` for time-based calculations.
- `DateTimeFormatter` provides safe and flexible formatting and parsing.
- Understanding time zones and choosing the correct date/time type is essential for building reliable enterprise and distributed applications.
