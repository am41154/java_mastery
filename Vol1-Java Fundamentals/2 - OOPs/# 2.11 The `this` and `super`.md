# 2.11 The `this` and `super` Keywords

### Why This Matters

As classes become larger and inheritance is introduced, it becomes important to distinguish between:

- The current object.
- Its parent object.
- Constructor parameters.
- Instance variables.

Java provides two special keywords for this purpose:

- `this`
- `super`

Understanding when and why to use them is essential for writing clean, maintainable object-oriented code.

---

# The `this` Keyword

The `this` keyword refers to the **current object**.

Whenever a method or constructor executes,

`this` represents the instance on which it was invoked.

Example:

```java
class Employee {

    private String name;

    Employee(String name) {
        this.name = name;
    }

}
```

Here:

```
name

↓

Constructor Parameter
```

```
this.name

↓

Instance Variable
```

Without `this`,

Java cannot distinguish between the parameter and the field.

---

# Calling Methods with `this`

The current object can invoke its own methods.

Example:

```java
class Employee {

    void print() {
        System.out.println("Employee");
    }

    void display() {
        this.print();
    }

}
```

The `this` keyword is optional here.

The following is equivalent:

```java
print();
```

However,

using `this` sometimes improves readability.

---

# Passing the Current Object

Sometimes an object needs to pass itself to another method.

Example:

```java
class Employee {

    void register() {
        Database.save(this);
    }

}
```

Here,

the current object is passed as an argument.

This is common in callback APIs and fluent builders.

---

# Returning the Current Object

The `this` keyword is frequently used in method chaining.

Example:

```java
class Employee {

    Employee setName(String name) {
        this.name = name;
        return this;
    }

}
```

Usage:

```java
employee
    .setName("Rahul")
    .setAge(30)
    .setDepartment("IT");
```

This pattern creates clean, fluent APIs.

---

# Constructor Chaining with `this()`

One constructor can invoke another constructor within the same class.

Example:

```java
class Employee {

    Employee() {
        this("Unknown");
    }

    Employee(String name) {
        this.name = name;
    }

}
```

Execution:

```
Employee()

↓

Employee(String)

↓

Object Ready
```

Rules:

- `this(...)` must be the first statement.
- Only one constructor can be invoked.

---

# The `super` Keyword

The `super` keyword refers to the **immediate parent class**.

It is commonly used to:

- Invoke parent constructors.
- Invoke overridden methods.
- Access hidden parent fields.

---

# Calling Parent Constructors

Example:

```java
class Vehicle {

    Vehicle() {
        System.out.println("Vehicle");
    }

}

class Car extends Vehicle {

    Car() {
        super();
        System.out.println("Car");
    }

}
```

Output:

```
Vehicle

Car
```

If `super()` is omitted,

the compiler inserts it automatically (provided the parent has a no-argument constructor).

---

# Calling Parent Methods

Example:

```java
class Vehicle {

    void start() {
        System.out.println("Vehicle Started");
    }

}

class Car extends Vehicle {

    @Override
    void start() {

        super.start();

        System.out.println("Car Started");

    }

}
```

Output:

```
Vehicle Started

Car Started
```

The child extends,

rather than replaces,

the parent's behavior.

---

# Accessing Parent Fields

Although less common,

`super` can access hidden fields.

Example:

```java
class Vehicle {

    String type = "Vehicle";

}

class Car extends Vehicle {

    String type = "Car";

    void print() {

        System.out.println(super.type);

    }

}
```

Output:

```
Vehicle
```

Field hiding should generally be avoided,

as it reduces readability.

---

# `this` vs `super`

| `this` | `super` |
|----------|----------|
| Refers to current object | Refers to immediate parent |
| Accesses current fields and methods | Accesses parent fields and methods |
| Calls another constructor using `this()` | Calls parent constructor using `super()` |
| Used in fluent APIs | Used in inheritance |

---

# Constructor Execution Order

Consider:

```java
class A {

    A() {
        System.out.println("A");
    }

}

class B extends A {

    B() {
        System.out.println("B");
    }

}
```

Execution:

```
new B()

↓

A()

↓

B()
```

The parent constructor always executes before the child constructor.

This guarantees proper object initialization.

---

# Spring Boot Perspective

Although developers rarely write `super` in Spring applications,

they frequently use constructor chaining and `this` indirectly.

Example:

```java
@Service
public class OrderService {

    private final PaymentService paymentService;

    public OrderService(PaymentService paymentService) {
        this.paymentService = paymentService;
    }

}
```

The `this` keyword clearly distinguishes constructor parameters from instance variables.

This pattern is recommended for constructor injection.

---

# Production Perspective

Suppose a logging framework provides a base class:

```java
abstract class BaseService {

    void log() {

    }

}
```

A subclass overrides a method:

```java
@Override
void process() {

    super.log();

    // Business logic

}
```

The child preserves the common logging behavior while adding its own processing.

# Interview Focus

### Question

What is the difference between `this` and `super`?

### Strong Senior-Level Answer

> `this` refers to the current object and is used to access its fields, methods, or constructors. `super` refers to the immediate parent class and is used to invoke parent constructors, access parent methods, or reference hidden fields. Both keywords help distinguish object context, especially in inheritance hierarchies.

---

# Common Mistakes

❌ Forgetting that `this()` and `super()` must be the first statement in a constructor.

❌ Calling both `this()` and `super()` in the same constructor.

❌ Confusing object references with parent references.

❌ Overusing field hiding.

---

# Engineering Insight

A practical guideline is:

- Use `this` when working with the current object's state.
- Use `super` only when extending or preserving parent behavior.

In well-designed systems,

explicit `super` calls are relatively uncommon because parent classes generally expose behavior through well-defined methods rather than requiring subclasses to manipulate internal details.

---

# Engineering Takeaways

- `this` refers to the current object.
- `super` refers to the immediate parent.
- Constructor chaining improves code reuse.
- Parent constructors execute before child constructors.
- Use `super` only when extending parent behavior intentionally.

---

# Quick Revision

✓ `this` → current object.

✓ `super` → parent object.

✓ `this()` calls another constructor.

✓ `super()` calls the parent constructor.

✓ Parent constructors execute first.

# Senior Interview Challenge

Review the following code:

```java
class Vehicle {

    Vehicle(String type) {
        System.out.println(type);
    }

}

class Car extends Vehicle {

    Car() {

    }

}
```

### Will this code compile? Why or why not?

**Expected Discussion Points**

- It will **not** compile.
- The compiler inserts `super()` automatically.
- The parent class does not have a no-argument constructor.
- The child constructor must explicitly call:

```java
super("Car");
```

- Understanding constructor chaining is essential when designing inheritance hierarchies.

Senior engineers recognize constructor initialization issues immediately because they understand the object creation sequence rather than relying on compiler errors.
